# Coffee Shop Simulator

Автор: Жильцов Никита Владимирович

## Цель проекта

Создать симулятор кофейни, который позволяет пользователям управлять виртуальной кофейней, принимать заказы, готовить напитки и обслуживать клиентов.

## Общая арихтектура

1. Сервер Coffee Shop
   - Обрабатывает запросы от клиентов
   - Управляет состоянием заказов
   - Делегирует приготовление напитков асинхронным потокам-бариста

2. Клиент
   - Отправляет запросы к кофейне
   - Может создавать заказы, оплачивать, получать список напитков и проверять статус

## Функциональные требования

1. Управление напитками
    - Сервер должен предоставлять возможность получения списка возможных напитков
    - Должен быть использован паттерн **Factory** для создания объектов напитков
2. Создание и обработка заказов
    - Клиент может создать заказ
    - Заказу назначается уникальный номер **orderId**, он попадает в общую очередь напитков
    - Из очереди заказы берут бариста - асинхронные процессы, которые блокируются на время приготовления каждого напитка
3. Оплата заказа
    - При создании заказа клиент должен выбрать способ оплаты
    - Должны поддерживаться разные стратегии оплаты
4. Уведомление о статусе заказа
    - Должен реализовываться паттерн **Observer** для уведомления клиентов о статусе их заказов
    - Для каждого заказа есть статус его выполнения
    - Изменения статуса заказа отправляется всем подписанным наблюдателям: клиенту, логгеру
5. Асинхронная обработка заказов
    - При старте создается N бариста - асинхронных потоков
    - Выполнение заказа эмулируется sleep
6. Обмен сообщениями
    - Допускается как HTTP запросы, так и использование сокетов для обмена сообщениями между клиентом и сервером

## Нефункциональные требования

1. Необходимо логировать создание всех заказов и изменение их статусов
2. Сервер должен корректно обрабатывать несколько одновременных подключений клиентов

## Требования к тестированию

1. Юнит-тесты
    - Тестирование фабрики напитков
    - Тестирование стратегий оплаты
    - Тестирование логики обработки заказов
2. Mock тестирование
    - Тестирование компонентов по отдельности при мокировании других
3. Тестирование многопоточности
    - Проверка корректной работы бариста при одновременной обработке нескольких заказов

## Используемые темы курса

- Паттерны проектирования: Factory, Strategy, Observer
- Многопоточность и асинхронное программирование
- Сетевое программирование
- Тестирование: юнит-тесты, mock-объекты, тестирование многопоточности
